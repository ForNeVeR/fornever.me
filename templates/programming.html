Эпический код

Регулярные выражения в C++-коде выглядят очень страшно. Итак, code of the day (взято из последней версии cthulhu-bot):

const wregex quoted_arg(L"^\"(.*?[^\\\\])\".*");

А в новой версии (от 29.01.2010) эта строка стала выглядеть ещё страшнее:

const wregex quoted_arg(L"^\"(.*?(?<!\\\\)(\\\\\\\\)*)\".*");

Все эти ужасы связаны со спецификой применения регулярных выражений в C++: если слеш должен быть частью регулярного выражения "как есть" (т.е. остаться слешем) - его нужно экранировать при передаче внутрь регулярного выражения. А поскольку для этого нужно передать два слеша в соответствующей строке - каждый из них приходится экранировать ещё раз. В стандарте C++0x эта проблема слегка упрощается за счёт введения синтаксиса т.н. raw strings в виде r"string \ literal" (что аналогично "string \\ literal" в текущей нотации). Это позволит в будущем сократить количество экранирующих символов.
